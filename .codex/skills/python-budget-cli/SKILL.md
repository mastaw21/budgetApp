# python-budget-cli

## 목적
이 프로젝트에서 CSV 기반 가계부 CLI 앱을 개발할 때 사용하는 프로젝트 전용 안내 스킬이다.

## 핵심 원칙
- 테스트 우선으로 개발한다.
- 함수는 작고 단순하게 유지한다.
- 타입 힌트를 생략하지 않는다.
- CSV 구조와 데이터 변환을 먼저 명확히 한 뒤 구현한다.

## 작업 순서
1. 테스트를 작성한다.
2. 실패를 확인한다.
3. 최소 구현을 추가한다.
4. `pytest`와 `radon cc`로 품질을 확인한다.
5. 필요하면 `qa_engineer` 검수 결과를 반영한다.
